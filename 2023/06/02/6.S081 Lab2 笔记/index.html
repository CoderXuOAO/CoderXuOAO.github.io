xv6的特点

  • “SMP”(shared Memory Multiprocessor) : xv6支持多核，多核共享同一个主存，每一个核支持一个线程控制（但在一些高性能处理器上，会有单个核支持多线程控制）。
  • 设备：UART, one disk drive, Timer Interrupts, PLIC(platform level interrupt controller), CLINT(core local interrupt controller)。
  • 内存管理：每页4KB大小，内存分配使用单个空闲列表(single free list)，不可分配可变大小的内存，三级页表。
  • 进程调度：时间片调度算法（固定时隙为1,000,000cyc；所有核共享同一个就绪队列）。
  • boot sequence：在qemu上运行，跳过所有boot操作，没有bootloader，直接加载内核至0x80000000。
  • 锁：Spin Locks，sleep()，wakeup() 。
  • 用户地址空间：
    • 栈：在xv6的用户进程空间中用户栈的位置如下所示，大小为一页，在低于stack地址的部分还有一页GUARD PAGE（图中未画出），stack从高地址向低地址增长，栈的容量不会增长，固定为一页，GUARD PAGE用于判断user stack是否溢出，若溢出则进程会直接abort处理。而传统的Linux的用户栈处于较高的地址空间部分，由下往上增长，而堆位于低于栈的地址空间，由下往上增长，当二者相遇时为满。
    • trampoline为内核代码及数据区，所有用户进程的trampoline page映射到同一个物理页。
    • trapframe用来存储进程进入内核态时的寄存器环境，以便在退出内核态时可以恢复用户进程。不同进程的trapframe映射到不同的物理页。
    • 虽然RISC-V为64位宽的，但硬件只使用低39位，而xv6只使用其中的低38位，MAXVA = 238 - 1，但物理内存只有128MB。
    • xv6中C语言main的变量只有argc和*argv[]，没有环境变量arge，argc和*argv[]会在程序运行前放入stack中。
pic1:xv6的用户地址空间
  • RISC-V虚拟内存：xv6使用sv39架构，其使用三级页表，硬件只用低39位，xv6系统只使用其中低38位。

kernel部分代码解读

entry.s

xv6从entry.s开始启动，这个时候没有内存分页，没有隔离性，其为每一个cpu初始化一个stack0。

start.c

  • entry.s将控制转移到start.c中的void start()，start()函数将完成从机器模式到内核模式的切换，以及切换到内核模式前的一些准备工作，切换由mret指令完成。

RISCV机器的PMP(physical memory protection)将物理内存分为16个区域，pmpcfg用来设置相应区域的一些全局设置，pmpaddr用来指明上下界。详细可参考PolarFire® SoC MSS Technical Reference Manual (microchip.com)

mret指令将像ret一样返回，由于start.c已经设置了进入内核的准备事项，因此mret执行后，机器将从机器模式转为内核模式并进入main.c的main处开始执行。

main.c

cpuid为0的进行一些系统初始化的操作，其他cpu等待cpu0初始化后，再进行对其自身的一些初始化。

部分初始化不可调整顺序，其中__sync_synchronize()用来阻止编译器进行一些优化，这条指令的会告诉编译器，完成这条指令之前的所有指令之后，再进行后面的指令。通过这种方式可以确保cpu0将所有初始化完成后，再将started置为1（started变量由volatile声明，告诉编译器不要对此变量的存取进行优化，每次访问此变量的时候都应该从内存中读取该变量）。其他cpu会再while处等到cpu0完成所有初始化并置started变量为1。

再main.c中cpu0进行的最后一个初始化为userinit()，他会创建第一个用户进程，这种进程有点像胶水代码/Glue code(胶水代码不实现具体的功能，只是为了适配不同的部分而存在)，我们总是需要有一个用户进程在运行，这样才能实现与操作系统的交互，所以这里需要一个小程序来初始化第一个用户进程，该程序为下图二进制部分，这个进程又会exec init.c。init.c会fork并exec shell，之后无限循环。
